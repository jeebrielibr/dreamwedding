# Sprint Project Wedding organizer
## Description
Sprint ini mengikuti jadwal yang ditentukan dosen dalam matkul pemrograman fullstack.

# Sprint 1
- Setiap kelompok yang sudah terbentuk menentukan judul project
- Ketentuan project harus ada backend dan frontend yang dibangun dari awal
- Tentukan fitur apa saja yang akan ada di project yang akan dikembangkan
- Setiap anggota kelompok minimal membangun satu fitur didalam project untuk backend dan frontend
- Masukan pembagian task/backlog sesuai dengan tugas di template sprint yang sudah disediakan
- Kumpulkan dalam bentuk laporan yang sudah dibuatkan formatnya didalam elena
- Kumpulkan laporan ke asisten dosen dan elena

# Sprint 2
- Setiap kelompok memastikan judul project yang akan dikembangkan sudah final
- Setiap kelompok memastikan daftar fitur yang akan dikembangkan pada project sudah ditentukan
- Setiap anggota kelompok memastikan fitur yang menjadi tanggung jawabnya sudah jelas
- Melakukan instalasi dan setup project backend menggunakan Express.js
- Backend harus sudah dapat dijalankan dan menampilkan endpoint sederhana
- Membuat struktur folder dasar project backend menggunakan Express.js
- Setiap kelompok mulai melakukan perancangan database yang akan digunakan pada project
- Menentukan tabel, field, dan relasi yang akan digunakan pada database
- Membuat desain database dalam bentuk ERD atau diagram database
- Masukan progres pengerjaan kedalam pembagian task/backlog sesuai template sprint yang sudah disediakan
- Kumpulkan laporan sesuai format yang sudah disediakan di Elena
- Kumpulkan laporan kepada asisten dosen dan Elena
- Buat repository kelompok dan collaborate akun github masing masing asdos dan akun arip86

## Sprint 5
Masuk Sprint minggu ke-5
- Setiap kelompok melanjutkan pengembangan backend dengan fokus pada fitur Authentication & Authorization
- Mengimplementasikan fitur register dan login user menggunakan Express.js
- Menggunakan bcrypt untuk hashing password saat proses register
- Mengimplementasikan JWT (JSON Web Token) untuk proses authentication
- Membuat middleware authentication untuk melindungi endpoint (protected route)
- Menambahkan middleware authorization (role-based access) untuk membatasi akses user (admin / user)
- Memastikan hanya user yang memiliki token valid yang dapat mengakses endpoint tertentu
- Mengatur sistem role (misalnya: admin dan user) pada data user
- Melakukan testing endpoint authentication (register, login, protected route) menggunakan Postman
- Melakukan testing authorization untuk memastikan pembatasan akses berjalan dengan benar
- Setiap anggota melanjutkan pengerjaan fitur sesuai pembagian tugas
- Memperbarui progres pengerjaan pada backlog / task sprint
- Kumpulkan laporan sesuai format yang sudah disediakan di Elena
- Kumpulkan laporan kepada asisten dosen dan Elena
